静态博客Hexo新手攻略

node.js知道么？不知道也没关系，只要你愿意折腾，你就可以拥有你自己的Github Pages。

第一步：安装

你需要首先装好node，当然不会少了npm。

好了，说一下我的安装环境吧，我使用的软件版本如下：

Win8.1 64bit
node v0.10.24
npm 1.3.21

讲完环境，咱开始正式安装，不用害怕，不会太难。什么？不会安装node？npm不会配环境变量？滚粗！！！

npm install -g hexo

我说过很简单吧？就这样就ok了！好，目前我的Hexo的版本是2.8.3，不一样没问题，给你看看我的版本～

然后就是在命令行中定位到你要部署博客的地址，愣着干嘛，创建项目啊：

D:\>hexo init blog
[info] Copying data
[info] You are almost done! Don't forget to run `npm install` before you start blogging with Hexo!

啧啧，你看看人家的代码写的多么猿类友好，在你用他爽之前别忘了运行npm install命令，你看着我干嘛，直接运行啊：

D:\>cd blog
D:\blog>npm install
npm http GET http://registry.cnpmjs.org/hexo-renderer-stylus
npm http GET http://registry.cnpmjs.org/hexo-renderer-ejs
npm http GET http://registry.cnpmjs.org/hexo-renderer-marked
npm http 200 http://registry.cnpmjs.org/hexo-renderer-ejs
npm http 200 http://registry.cnpmjs.org/hexo-renderer-marked
......

然后就装好了，问我怎么运行，看着：

D:\blog>hexo server
[info] Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.

尝试着在浏览器里访问吧，是不是略屌？

第二步：配置

不要被这么高大上的名词迷惑，就是改改博客名字之类的。额…不过首先你得知道配置文件在哪。

配置文件在项目根目录，那个_config.yml就是它了！

title: 吉它电台
subtitle: 不想做产品经理的程序员不是好吉他手
description: 对互联网产品的跪舔和吐槽，码农一些踩坑的经历，以及闷骚吉他手对于生活的感悟
author: 祭塔封魔
email: xxxxxx@xxx.com
language: zh-CN

url: http://blog.jitafm.com/

你们牛逼的肯定知道该怎么改，和我一样二货的就改上面的东西就行了，对了下面还有一个URL地址，你们看着填，英文懂吗？跟我读A…B…C…

第三步：开始装逼

装什么逼？写日志啊！！！

打开另一个命令行，原来那个在运行server呢！别关掉了！

D:\blog>hexo new 你好人类
[info] File created at D:\blog\source\_posts\你好人类.md

你问我md是什么？Markdown你不知道？卧槽，那你写个毛博客啊，回家带孩子去吧。

如果你还没关掉网页的话，你应该知道再哪找到这个md文件，也应该知道如何打开它吧。

好，很好，注意打开后上面几行留着，这个是写个Hexo看的，照着我来改吧：

title: 静态博客Hexo新手攻略
date: 2014-11-06 16:26:11
tags:
- Hello World
- Hexo
- 静态博客
- GitHub Pages
- node.js博客
categories:
- 日志
- 程序员
---

好的，后面的东西你自己发挥咯，看到这里还是友情提示一下吧：Markdown语法不会Win下可以用MarkdownPad，使用起来挺简单的，适合新手玩玩，老鸟忽略…

对了，如果要插入图片，你在source目录下建个文件夹脚img，然后和我一起马克党：

第四步：生成文章

就这一步了，生成静态的文件，cmd输入：

D:\blog>hexo generate

然后呢，本地就多了一个public的目录，里面装着满满的逼格。

第五步：发布到GitHub

简单的说就是你在github上先创个项目，然后把_config.yml中的deploy相中填写好：

# Deployment
## Docs: http://hexo.io/docs/deployment.html
deploy:
  type: github
  repo: git@github.com:TingSky/blog.git

额，这个就别跟我填了，我没提交。

什么？你问我xxx.git是什么？额…GitHub用过吧？没用过的您都看到这了，去建一个吧，那逼格比我这高。

好了，然后就提交啦！打开你的 Git Bash，输入下面的命令，一定要打开Git Bash :

hexo deploy

这个你刚刚建立的静态网站就被部署到了GitHub上了，检查一下分支是gh-pages。gh-pages是GitHub为了web项目特别设置的分支。

然后点Settings，你就可以得到你的GitHub Pages地址啦。

赶快开始装逼吧！！！
